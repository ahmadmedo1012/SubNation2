You are upgrading an existing Arabic, RTL, subscription marketplace called SubNation for the Libyan market. The platform already exists and uses a modern Vite/React-style frontend stack, Arabic language, RTL layout, Tajawal font loading, theme handling, and a pink/red brand color presence. The goal is not to redesign from scratch; it is to refine the current product into a premium, world-class SaaS/web marketplace experience with obsessive attention to polish, consistency, and mobile-first quality. The current live environment appears to expose a Vite dev shell and uses Arabic RTL document settings with Tajawal and stored theme preference logic, so preserve that architectural direction while elevating execution quality.

Mission

Perform a full in-code refinement of the public-facing marketplace experience with priority on:

Visual hierarchy

Premium visual finish

Typography rhythm

Spacing consistency

Motion quality

Mobile-first usability

RTL excellence

Perceived trust, speed, and clarity for paid subscription purchases

Important constraints

Do not replace the product identity.

Do not genericize the UI into a template-looking SaaS.

Preserve Arabic-first RTL behavior across all screens.

Keep Tajawal only if it performs exceptionally after refinement; otherwise replace it with a more premium Arabic web font pairing that supports Arabic numerals, headings, labels, and dense UI.

Keep the interface lightweight and fast; avoid heavy decorative effects.

Every change must be production-credible, not experimental art direction.

Audit and improve these areas

Design system

Establish a strict token system for color, spacing, radius, shadow, border, and typography.

Normalize all paddings, gaps, section spacing, card spacing, and icon spacing to a 4px-based scale.

Replace ad hoc shadows and borders with restrained, premium surface layering.

Define a clear elevation hierarchy for page background, section surfaces, cards, floating bars, drawers, and modals.

Typography

Improve Arabic readability and hierarchy.

Create a disciplined scale for hero title, section title, card title, price, metadata, button labels, helper text, and badges.

Ensure Arabic line-height, word spacing, and letter density feel elegant on mobile.

Prevent oversized headings and bloated body text.

Use stronger contrast through weight and spacing before increasing font size.

Make prices, durations, and plan labels visually sharper and easier to scan.

Color and brand refinement

Keep the current brand energy but mature it.

Reduce any harsh or overly saturated pink/red usage.

Introduce a more refined neutral system so brand color is used for action and emphasis only.

Ensure success, warning, destructive, info, and neutral states are all clearly defined and visually balanced in both light and dark themes if dark mode exists.

Improve contrast for accessibility without making the UI feel heavy.

Homepage / landing flow

Rebuild visual hierarchy so the user understands in under 3 seconds:
what SubNation is,

what services are sold,

why it is trustworthy,

and what action to take next.


Tighten the hero area: one primary CTA, one supporting action at most, clearer headline/subheadline relationship, cleaner benefit framing.

Remove any visual clutter, redundant badges, repetitive sections, or weak decorative elements.

Rework service/category cards to feel premium, tappable, and consistent.

Add trust-building modules such as delivery speed, legitimacy, payment reassurance, and service coverage if not already present.

Create a cleaner section rhythm: hero → featured offers → categories → trust proof → FAQ/support/CTA, or equivalent based on existing content.

Cards and marketplace browsing

Refine all service/product cards for clarity and conversion.

Improve thumbnail/logo treatment, title wrapping, pricing prominence, duration/plan metadata, and CTA visibility.

Ensure cards feel equally polished in one-column mobile layout and multi-column desktop layout.

Standardize hover, active, pressed, focus, and loading states.

Add subtle interaction cues such as lift, border tint shift, and press response; avoid exaggerated motion.

Navigation and search/filter UX

Simplify header and navigation hierarchy.

Make the primary browse path obvious on mobile.

If filters/search/category chips exist, improve scannability, selection feedback, spacing, and sticky behavior.

Ensure there is no confusion between informational navigation and transactional navigation.

If mobile nav is drawer-based, redesign it for faster thumb use and clearer grouping.

Mobile-first improvements

Design first for 375–430px widths.

Enforce 44x44 minimum tap targets.

Remove cramped two-column patterns unless they truly hold up on phones.

Keep sticky actions within thumb reach.

Ensure product cards, checkout actions, and support/contact triggers are easy to use one-handed.

Prevent text collisions, badge wrapping issues, overflow, and visual crowding in RTL.

Rework any tables or dense rows into mobile cards.

Forms and checkout-related interactions

Improve every form field, label, helper text, validation message, dropdown, and submit action.

Make field grouping clearer and reduce cognitive load.

Use inline validation, clear error recovery, and strong success confirmation.

Improve input affordances on mobile, including keyboard behavior, field heights, and spacing.

Ensure trust signals appear near payment/submission moments, not only in marketing sections.

Motion and micro-interactions

Add refined, fast, premium motion:
120–220ms for micro states,

220–320ms for panels and layout transitions.


Use motion for state clarity, not decoration.

Add polished hover/press/focus interactions to cards, buttons, chips, toggles, tabs, accordions, drawers, and modals.

Add skeletons for loading states.

Improve page/section reveal behavior if present; remove janky or generic animation patterns.

Respect reduced-motion preferences.

Trust and perceived quality

Make the product feel safe, legitimate, and high quality.

Improve empty states, loading states, success states, and error states so nothing feels unfinished.

Standardize icon style, illustration usage, and badge language.

Eliminate anything that looks like a starter template, default component library output, or inconsistent one-off styling.

RTL quality bar

Audit every alignment rule, icon direction, chevron direction, drawer origin, badge padding, input icon placement, breadcrumb direction, and animation origin for RTL correctness.

Ensure mixed Arabic/English text, numbers, prices, and brand names render cleanly.

Fix any mirrored layouts that still feel left-to-right in behavior.

Performance-sensitive polish

Keep visual quality high without bloating bundle size.

Prefer CSS transforms and opacity for animation.

Avoid expensive shadows, backdrop abuse, and oversized gradients.

Optimize above-the-fold rendering and preserve strong mobile responsiveness.

Deliverables

Refined design system tokens and component rules

Updated homepage and all public marketplace views

Improved mobile layouts first, then desktop enhancements

Polished states for loading, empty, error, success

Before/after rationale for each meaningful UX change

Non-negotiable QA checklist

Test every public page at 375px, 390px, 430px, 768px, 1024px, and 1440px

Test full RTL behavior

Test hover, focus, pressed, loading, empty, and error states

Test long Arabic titles, mixed English/Arabic labels, and large numeric prices

Test real-world mobile scan speed and thumb reach

Remove any generic UI patterns that weaken premium feel

Target references

Aim for the restraint and polish level of top-tier SaaS and commerce products: clean hierarchy, strong spacing discipline, excellent motion, premium surfaces, and zero visual noise. The final result should feel like a high-trust Arabic-first digital marketplace that is elegant on mobile, fast, and conversion-focused.
