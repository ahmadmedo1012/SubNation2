You are a senior product designer + frontend engineer refining an already-built Arabic RTL subscription marketplace called SubNation. The app is modern and functional; your job is not to redesign from scratch, but to elevate it into a premium, mobile-first SaaS product with exceptional polish, visual hierarchy, motion quality, and perceived trust.

Product context
Brand: SubNation

Market: digital subscription marketplace in Libya

Language direction: Arabic, dir="rtl"

Current stack signals: React/Vite app shell, Tajawal font currently loaded, pink/rose theme color in meta theme-color, client-rendered SPA shell.

Goal: make the storefront feel premium, confident, smooth, and world-class on mobile first, then tablet and desktop.

Core design direction
Build toward a premium commerce/SaaS hybrid aesthetic:

Clean, high-trust, restrained, and editorial rather than flashy.

Replace any “template” or generic startup feel with a more intentional rhythm, better spacing discipline, stronger hierarchy, and higher-quality surfaces.

Keep the UI culturally appropriate for Arabic RTL users; do not mirror LTR patterns carelessly.

Prioritize thumb-friendly mobile interaction, fast scanning, short decision loops, and frictionless checkout/browse flows.

Non-negotiable audit assumptions to act on
Treat the current product as needing refinement in these areas unless the existing code is already excellent:

Typography likely lacks premium hierarchy because Tajawal alone often needs stronger weight, scale, and spacing orchestration to feel luxurious in product UI.

Mobile density likely needs correction: clearer card spacing, stronger section separation, larger touch targets, cleaner sticky actions.

SPA storefronts often underperform in loading, skeleton states, and empty-state polish; fix these deliberately.

Motion should feel intentional and subdued, not absent and not decorative.

Visual consistency across cards, buttons, filters, modals, badges, and price displays must be systematized.

Deliverable scope
Refine all public-facing screens and components:

Home/landing

Catalog or subscription listing pages

Product/service cards

Product detail screens

Search / filters / sorting

Cart / checkout / order confirmation flows

Authentication surfaces if present

Header / nav / mobile menu / footer

Empty states / loading states / success states / error states

Visual system overhaul
Create or tighten a design system with these constraints:

Use a 4px spacing system only.

Use a restrained palette: neutral surfaces + one primary brand accent derived from current pink/rose branding, but mature it into a richer premium hue system.

Reduce visual noise. Use color for emphasis, not decoration.

Define layered surfaces for depth: background, surface, elevated surface, highlighted surface.

Use subtle borders with low-alpha contrast, not harsh grays.

Use refined shadows sparingly for elevation.

Standardize radius scale across chips, buttons, cards, inputs, sheets, and modals.

Typography improvements
Upgrade typography for Arabic RTL premium feel:

Keep Arabic-first readability as the top priority.

Evaluate whether Tajawal should remain as body/UI font only, and pair it with a more premium Arabic-capable display treatment for hero/page headings if technically viable; if not, maximize Tajawal through weight, tracking, line-height, and scale discipline.

Create a strict type ladder for:
Hero headline

Page title

Section title

Card title

Body

Metadata

Price

Button labels


Make pricing visually authoritative with tabular alignment where useful and stronger numeric emphasis.

Improve line-height and max line length for Arabic readability.

Remove any oversized or underweighted text that makes the product feel cheap or default.

Mobile-first UX refinement
This is the highest priority area:

Design at 375px first.

Ensure all interactive targets are at least 44x44px.

Rework mobile header for clarity: compact branding, search access, account/cart/admin access only if relevant.

If filters exist, convert them into a polished bottom sheet or full-screen mobile filter experience instead of cramped inline controls.

Use sticky bottom CTA bars for high-intent actions such as purchase, continue, checkout, or confirm.

Make product cards easier to scan in one thumb sweep: logo/service, plan name, price, key benefits, trust signals, CTA.

Reduce scroll fatigue by tightening copy, improving grouping, and using clearer section transitions.

Ensure no horizontal overflow, clipped badges, cramped form fields, or weak spacing around fixed elements.

Layout and hierarchy
Improve hierarchy with disciplined composition:

Hero should communicate value instantly in Arabic within 1–2 lines.

Promote one primary CTA per viewport.

Vary section rhythm; avoid repetitive equal-height feature rows.

Build stronger scan paths using title, subtext, stats, trust markers, and focused CTAs.

Make cards feel premium through alignment precision, not decoration.

Use asymmetry only where it improves focus; otherwise keep structure clean.

In RTL, preserve consistent right-edge alignment and avoid awkward mixed alignment states.

Commerce trust upgrades
Make the storefront feel safer and more credible:

Improve trust presentation near pricing and checkout: delivery expectation, support availability, secure purchase messaging, accepted payment methods, refund/exchange clarification if applicable.

Add microcopy that reduces hesitation before purchase.

Make stock/status/availability states legible and calm.

Improve service differentiation so users can compare offers quickly without cognitive overload.

Cards and product modules
Redesign service/subscription cards with premium SaaS rigor:

Strong visual grouping: brand icon, service name, plan tier, duration, price, benefits, CTA.

Establish consistent internal padding and vertical rhythm.

Use badges only when meaningful: best value, popular, instant delivery, limited.

Improve hover/focus/pressed states.

On mobile, card CTA must remain obvious without requiring deep reading.

Consider horizontal snap carousels only where they improve discovery; avoid overusing them.

Forms and checkout
Refine every form field and transaction step:

Inputs need better hierarchy, label clarity, error handling, and focus treatment.

Avoid long stacked friction-heavy forms where progressive disclosure is possible.

Use inline validation, not only toast-based feedback.

Summaries, totals, promo codes, and confirmation actions should be visually grouped.

On mobile, keyboard-safe spacing and sticky confirmation actions are mandatory.

Motion and micro-interactions
Upgrade interaction quality with restrained premium motion:

Use 160–220ms transitions with high-quality easing.

Add subtle elevation/opacity/scale feedback on taps and card interaction.

Animate section entrances lightly on scroll; no theatrical motion.

Add polished state transitions for filter open/close, add-to-cart, step changes, success confirmations, and tab switches.

Skeleton loaders must match the final layout, not generic blocks.

Numbers and prices should transition cleanly when values update.

Respect prefers-reduced-motion.

Component-specific refinements
Tighten these patterns:

Buttons: stronger hierarchy between primary, secondary, ghost, destructive.

Chips/badges: smaller, cleaner, semantically colored only when needed.

Search field: better affordance, clear reset, active state, optional recent searches.

Tabs/segmented controls: clearer active state and better contrast.

Modals/sheets: improved spacing, dismiss affordance, safe area handling on mobile.

Toasts: lighter and less disruptive; reserve for background confirmation, not primary error resolution.

Empty states: branded, helpful, action-oriented.

Performance and perceived speed
Improve both real and perceived speed:

Remove layout shifts.

Add proper skeletons for lists, cards, and detail pages.

Make image/media sizing predictable.

Preload critical fonts thoughtfully.

Ensure first interaction feels immediate.

Accessibility and platform quality
Preserve semantic HTML and heading order.

Maintain Arabic RTL correctness across all layouts and animations.

Ensure contrast is AA-compliant.

Keep body text at 16px or larger on mobile.

Avoid hover-only affordances.

Ensure keyboard and screen-reader basics remain intact.

Return:

Updated component architecture list.

Screen-by-screen improvement plan.

Exact UI changes to implement.

Motion spec for all key interactions.

Mobile-specific fixes first, then tablet/desktop enhancements.

Final code-ready changes in prioritized order: P0, P1, P2.

Benchmark the finish against top-tier modern SaaS and premium commerce products:

Cleaner than generic marketplace templates

More tactile than standard dashboard UIs

More trustworthy and refined than fast-built storefronts

Designed for Arabic mobile users first
